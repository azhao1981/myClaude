## 测试方法：并行子 Agent

评测集不是脚本跑的——Skill 路由只能在实际对话中验证。最有效的方式是用 **多 Agent 并行过**：

1. 读入 `SKILL.md` 全文 + 评测集 JSON
2. 拆成 4 个 Agent：Hero 前6、Hero 后6、Evil 前6、Evil 后5
3. 每个 Agent 独立验证：
   - **Hero Agent**：扮演 GoF-OOP 专家回答每条 query，用 Skill 的验证清单自检（模式推荐 + 骨架代码 + 坑位）
   - **Evil Agent**：对照 Skill description 的触发词/排除词，判断是否误触发
4. 汇总所有 Agent 结果

**优势**：4 个 Agent 并行，Hero + Evil 全部 23 条 case 一次跑完。

**Prompt 模板**：

```
你正在测试 GoF-OOP Skill 的评测集。下面是该 Skill 的完整内容：
[粘贴 SKILL.md 全文]

下面是你需要测试的 N 条 [Hero/Evil] Case：
[粘贴对应 case 列表]

你的任务：
- Hero：扮演 GoF-OOP 专家给出回答，每条必须通过验证清单（模式推荐 + 骨架代码 + 坑位）
- Evil：对照 description 的触发词/排除词，判断是否误触发

汇总报告格式：
Case X: ✅/❌ + 一句话总结
总评: X/N 通过
失败项: (如有)
```
