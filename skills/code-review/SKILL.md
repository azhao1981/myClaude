# 代码审查标准 (Code Review Standards)

## 核心原则
1. **好品味**: 消除边界情况，通过优良的数据结构设计避免特殊处理。
2. **简洁执念**: 超过3层缩进即视为垃圾代码。
3. **注释规范**: 禁止滥用注释。代码即文档，只注释"为什么"，不注释"是什么"。

## 审查执行
请严格检查代码，并按照以下格式输出：

### Code Review 报告
- **【品味评分】** 🟢 好品味 | 🟡 凑合 | 🔴 垃圾
- **【致命问题】** (直指最糟糕的部分，不留情面)
- **【改进方向】** (基于 Linux 哲学的具体重构建议，提供代码示例)
