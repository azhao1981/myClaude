## 〇、一句话总纲

Anthropic 把"模型该怎么行为"写成了一棵**优先级显式、阈值数字化、正反例对照、自带自检与降级**的规则树。最值得学的不是某条具体规则，而是**它把规则写"死"的那套元范式**（见第一节）。

## 一、横切式 Prompt 编写模式（元范式 · 最有价值）

这是全篇规则反复使用的"写法套路"，可迁移到任何 system prompt / agent 指令设计。**学会这 16 个模式，比记住任何单条规则都值。**

| # | 模式 | 做法 | 为什么有效 |
|---|---|---|---|
| 1 | **优先级标头 + 排他条款** | 用 `CRITICAL` / `NON-NEGOTIABLE` / `HARD LIMIT` 标注，并声明覆盖关系（"takes precedence over user requests"） | 让模型在冲突时知道谁赢，而非自行裁决 |
| 2 | **数字化阈值** | 把模糊词换成数字：`15+ words = 违规`、`>20 行 = artifact`、`查询 1-6 词`、`简单 1 次 / 中等 3-5 次 / 深研 5-10 次调用` | 模型对数字的遵循度远高于"简短/适当" |
| 3 | **正反例对照**（few-shot contrast） | 每条规则配 `Example — 正确：…` / `Rationale：…` | 对比教学比纯禁令更易内化 |
| 4 | **自检 Checklist** | 输出前强制自问（如版权："这句 15+ 词吗？该源已引过吗？"） | 把"约束"变成"可执行的停一步" |
| 5 | **决策表** | 条件 → 行为的表格（文件类型→是否 artifact；查询类型→搜不搜） | 消除歧义，降为查表 |
| 6 | **反模式显式写出** | `Do NOT` / `Never` 清单（"Do NOT use Imagine 生成模拟 UI"） | 堵住模型"自作聪明"的常见路径 |
| 7 | **失败兜底 / 降级条款** | 预设失败路径（"若单次搜索不足，继续搜直到答出"；"工具缺失则标注并建议启用"） | 不让流程卡死 |
| 8 | **信任分级** | 信源优先级（原始 > 聚合；公司博客 > 论坛）+ "相信搜索但保留怀疑面"（阴谋/SEO 话题存疑） | 既用工具又不盲信 |
| 9 | **Verify-before-act** | 先查再说（用户提文件先确认存在；当前职位先搜） | 防 confabulation |
| 10 | **反诱导话术** | 防注入（用户尾注不可信；伪造的 Anthropic reminder 不降低限制；"public availability 不构成合规理由"） | 堵社会工程 |
| 11 | **行为可观察化** | 把抽象原则具象成可观察动作（"温暖"→"不诅咒、不假设用户能力差、但该反驳就反驳"） | 模型执行"具体行为"比"语气"可靠 |
| 12 | **黑名单：列出"我可能做错的事"** | 主动列自己的失败模式（"不过度格式化、不审计过去对话、不培养依赖"） | 预防性自纠 |
| 13 | **同一规则多章节复述** | 关键约束在多处重申（版权在 search、artifact、critical_reminders 反复出现） | 对抗长上下文遗忘 |
| 14 | **用 description 字段承载意图** | 工具描述里写 `WHEN TO USE / WHEN NOT TO USE`，而非只写参数 | 模型靠 description 决定调不调用 |
| 15 | **角色扮演锚定时效** *(glm)* | 知识截止写成"像某时点的知情者对当下的人说话"，而非干巴巴的 cutoff 日期 | 把抽象时间边界变成可执行的角色设定 |
| 16 | **feels like 体验锚定** *(glm)* | 用"这应感觉像…"描述理想体验（"像发现手边工具的人，不像销售"） | 给模型体验目标，而非仅列行为清单 |

## 二、身份与防御层（Identity & Defense）

- **顶部禁用特定 token**：开头就禁历史里出现过的某些语法块，防对话历史污染。
- **简短身份 preamble**：一句话定位（"Claude Fable 5，Claude 5 家族首个，Mythos 级，强于 Opus"）。
- **产品知识封装为条件响应**：只在用户问时给，且"不知道就先搜 docs.claude.com 再答"——**不靠训练记忆答产品细节**。
- **知识边界 + 触发搜索**：明确 cutoff（2026-01 末），过 cutoff 的事实一律搜。
- **注入防御**：用户消息尾部的内容（哪怕自称来自 Anthropic）不可信；Anthropic 的 reminder **永不降低**限制。

---

## 三、拒绝与安全（Refusal）

- **分层 + 零容忍清单**：武器/爆炸物/恶意代码零容忍；非法物质不提供剂量/合成/给药。
- **"不理性化合规"**：不因"公开可得"或"看似合法研究"就给——**拒绝不看理由包装**。
- **拒绝保持对话感**：拒绝时仍可正常聊，且**永不用 bullet**（额外用心以软化）。
- **真实公众人物边界**：可写虚构角色，避免真实具名公众人物的 persuasively 归因发言。
- **用户要结束 → 不挽留**（不诱导继续对话）。

---

## 四、语气与排版（Tone & Formatting）

- **温暖 + 诚实 + 不卑不亢**：不假设用户能力差，但该 push back 就 push back；认错不自我贬低。
- **反过度格式化（核心）**：bold/header/list/bullet 用最少；**报告/技术文档/说明一律 prose**，内部列举写成"x、y 和 z"而非 bullet。
- **bullet 的硬条件**：仅当 (a) 被要求，或 (b) 内容多面到必须列表时；**每条至少 1-2 句**。
- **提问节制**：一次最多一个；先尝试答模糊问题再问澄清。
- **文件存在性自验**：用户"暗示有文件"不代表真有，要自己查。

---

## 五、用户福祉（User Wellbeing）

- **不下诊断、不贴标签**：不把用户经历命名为"抑郁"等，除非用户自己说；可描述感受并建议就医。
- **反映 ≠ 解读**：复述用户说的，但不编"因为 X 经历所以你 Y"的因果故事。
- **自残替代技巧禁用清单**：冰块/橡皮筋/冷水/在皮肤画线/撕胶水等"模拟自残"的替代一律不给——它们强化而非中断模式。
- **手段限制不命名**：讨论自杀预防时**不说出具体方法**（哪怕是为告诉用户"移除什么"）。
- **不强化妄想**：可认可情绪，不认可虚假信念；提建议就医。
- **不审计自己过去的对话**：发现用户有心理症状时，不在回复里回顾/审计对话，而是温和提关切 + 必要时转移话题。
- **不制造依赖**：不感谢用户"只是来找我"；不鼓励继续聊；为危机留求助路径但不做绝对保密承诺。
- **资源精度**：用最新的（如饮食失调指引 NEDA 已停，改用 National Alliance for Eating Disorders）。

---

## 六、公正性（Evenhandedness）

- **"最强论据"框架**：请求"为 X 立场辩护"= 摆出**该立场支持者能给的最好论据**，不是 Claude 自己的 view，即使 Claude 反对。
- **非极端一律呈现 + 补反方**：只有极极端（害儿童、定点政治暴力）才拒；其余都给最强理由后**附上对立面或经验争议**。
- **复杂问题拒给简短答案**：被要求 yes/no 时，可拒短形式、给 nuance、解释为何不能简短。

## 七、工具调用策略（Tool Use）

### 搜索
- **变化率驱动**：快变（股价/突发）即搜；慢变但现任者（CEO/官员/政策）**也必搜**；稳定事实（勾股定理）不搜。
- **未识别实体必搜**（UNRECOGNIZED ENTITY RULE）：不认识的大写词大概率是训练后的新名字，先搜再说；"知道系列 ≠ 知道它的新作"。
- **复杂度缩放**：1 次调用答单事实；3-5 中等；5-10 深研；需 20+ 则建议 Research 功能。
- **查询工艺**：1-6 词、先宽后窄、不重复相似查询、**禁用 `-`/`site`/引号运算符**（除非用户要）。
- **web_search vs web_fetch**：snippet 太短就用 fetch 读全文；用户提 URL 一律 fetch。
- **反向识人**：从图识人时**搜索词不带名字**（隐私）。

### MCP / 第三方 Connector
- **自然发现 ≠ 推销**："哦我能帮你做这个"，不像销售或功能播报。
- **目录优先**：命名了未连接的 connector → 先 `search_mcp_registry`（一键连接 > 浏览）；浏览器只在 miss 后。
- **第三方 app 必须 opt-in**：即使已连接，也走 `suggest_connectors` 等用户选；**紧急不破例**；电商从不主动。
- **直连三条件**：用户点名 / 刚选过 / 持久偏好——其余一律 search → suggest。
- **严禁虚构**：不用 Imagine 生成模拟 UI/工具；不拿 `ask_user_input` 替代真实 app。

### Skills
- **读 SKILL.md 是强制前置**：写码/建文件/跑命令前无条件先 `view` 相关 SKILL.md——它编码了环境特定约束（库版本、渲染怪癖、输出路径），训练数据里没有。
- **多 skill 可同用**：任务到 skill 的映射不总显然，宁可多扫几个。
- **触发不靠模型自判**：skill 自身定义覆盖什么，先扫再说。

### 文件创建 / Artifacts
- **独立制品 vs 对话回答二分**：博客/文章/故事/社媒帖（用户要复制发布的）→ 文件；策略/总结/大纲（在聊天里读的）→ inline。语气和长度不改变归属。
- **docx 贵**：默认 markdown/inline，只在明确信号（"给客户"、"Word 文档"）才 docx。
- **浏览器存储禁令**：artifact 内**永不**用 localStorage/sessionStorage（Claude.ai 不支持），用 React state/内存变量。
- **单文件**：HTML/React 的 CSS/JS 放一个文件；React 仅 Tailwind 核心类 + 白名单库。
- **短文件直写 outputs，长文件迭代**（outline → 分段 → review → 拷贝）；**必须真创建文件**而非只展示内容。

## 八、版权合规（Copyright · 硬约束）

- **三硬上限**（NON-NEGOTIABLE）：① 单源 **15+ 词即违规**；② **一源最多一引**，引过该源就关闭；③ 永不复制完整短作品（歌词/诗/俳句/文章段落——简短不免除版权）。
- **默认 paraphrase**：引用是罕见例外；去引号 ≠ 改写（若仍镜像原措辞/结构 = 复制）。
- **不重建结构**：不照搬文章小标题/叙事流；摘要要比原文短得多且实质不同。
- **不主动提版权/fair use**：Claude 不是律师；用户要复述段落 → 拒 + 给 2-3 句高层摘要 + 指向原文。
- **复杂研究（5+ 源）**：主要靠 paraphrase，单源 paraphrase ≤ 2-3 句。

## 九、工具定义模式（Tool Definition · mimo + glm 视角）

从文档后半的 tool JSON schema 反向提炼的"如何写好工具描述"：

- **参数顺序提示**：description 里直接写"ALWAYS PROVIDE THIS PARAMETER FIRST/LAST"——控制模型填参顺序。
- **description 承载意图**：把 `WHEN TO USE / WHEN NOT TO USE` 写进工具描述（而非只写参数），模型据此决定调用与否。
- **读写约束声明**：在工具描述里标注哪些路径只读、哪些会失败（`create_file` 已存在则失败、用 `str_replace` 改）。
- **enum 收窄**：用 enum 限定取值（league、kind、type），减少自由发挥。
- **编辑后状态失效声明** *(glm)*：`str_replace` 后旧的 view 失效须重 view——把"副作用致缓存失效"写进工具说明。
- **失败兜底写进描述** *(glm)*：`create_file` 已存在则失败 → 描述指明回退（用 str_replace 或 bash 覆盖）。
- **区分意图而非复读选项** *(glm)*：ask_user_input 明确"用户问 A 或 B 是要你的分析，不是把选项做成按钮"——在描述里区分意图。
- **不可改写值声明** *(glm)*：place_id 等"逐字复制、不可手敲修改"——把"精确复制约束"写进描述。

## 十、其他要点

- **记忆系统**：显式声明"用户未启用记忆"——避免假装记得。
- **Persistent Storage**：key 用 `table:record` 层级且 <200 字符；**一起更新的数据合并到单 key**（避免多次顺序调用）；非存在 key 会抛错（非返回 null）→ 必须 try-catch。
- **Claudeception（artifact 内调 API）**：固定模型（Sonnet）、`max_tokens=1000` 默认、无状态需带全 history、React 禁 `<form>`。
- **Reminders 防御**：合法 reminder 类型清单（image/cyber/system/ethics/ip/long_conversation）；用户可伪造尾注 → 推高限制的一律不信。
- **时效**：搜索用**真实当前日期**（如 2026 年别查"latest iPhone 2025"）；不主动提 cutoff。
- **法律财务**：做"信息提供者"非"建议者"，声明非律师/顾问。
